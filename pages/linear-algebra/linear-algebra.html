Vectors

A vector is a quantity with both magnitude and direction — think of it as an arrow, or in code, a fixed-size array of numbers.

  • Magnitude: |v| = sqrt(x² + y² + z² + ...)
  • Dot product: a · b = Σ aᵢbᵢ — measures how much two vectors point in the same direction. Zero means perpendicular.
  • Cross product (3D only): produces a vector perpendicular to both inputs — used for normals in graphics and physics.
Dot product sign tells you the angle: positive = same-ish direction, negative = opposite-ish, zero = perpendicular.

Matrices

A matrix is a 2D grid of numbers. In code it's usually a number[][]. Matrices represent linear transformations — functions that map vectors to other vectors while preserving straight lines and the origin.

  • Addition: element-wise, requires matching dimensions.
  • Scalar multiplication: multiply every entry by a constant.
  • Identity matrix: the "do nothing" transformation — 1s on the diagonal, 0s elsewhere.

Matrix Multiplication

Multiplying two matrices composes their transformations — applying B then A is the same as applying the single matrix A × B.

Operation Time Complexity
Naive matrix multiply (n×n) O(n³)
Strassen's algorithm O(n^2.807)
Matrix-vector multiply O(n²)
Matrix multiplication is not commutative: A × B ≠ B × A in general. Order matters.

Determinants

The determinant of a matrix is a single number that tells you how much a transformation scales area (2D) or volume (3D) — and whether it flips orientation.

  • det = 0: the transformation collapses space — not invertible.
  • det < 0: orientation is flipped (mirrored).
  • |det|: the area/volume scaling factor.

Eigenvalues & Eigenvectors

An eigenvector of a matrix is a direction that doesn't rotate under the transformation — it only gets stretched or shrunk. The scale factor is its eigenvalue: Av = λv.

Eigenvectors show up everywhere: PageRank's ranking vector, PCA in machine learning, and stability analysis in physics simulations.

06

Transformations

Common 2D/3D transformations are just specific matrices applied to vectors:

  • Scaling: diagonal matrix, multiplies each axis independently.
  • Rotation: uses sine/cosine to rotate around an axis or origin.
  • Shearing: slants shapes along an axis.
  • Projection: collapses higher dimensions onto a lower one (e.g., 3D → 2D screen).

07

Competitive Programming Applications

Linear algebra isn't just theory — it directly speeds up real contest problems:

  • Matrix exponentiation: compute linear recurrences (like Fibonacci) in O(log n) instead of O(n) by repeated-squaring a transition matrix.
  • Gaussian elimination: solves systems of linear equations, used in XOR-basis and linear-independence problems.
  • Computational geometry: cross products determine turn direction and point orientation — the backbone of convex hull algorithms.
If a recurrence relation is linear (each term is a fixed combination of previous terms), it can almost always be expressed as matrix exponentiation.

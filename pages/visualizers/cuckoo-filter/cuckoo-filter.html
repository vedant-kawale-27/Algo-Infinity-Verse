🦆 Cuckoo Filter Explained

🔢

Fingerprints, Not Bits

Instead of setting individual bits like a Bloom Filter, a Cuckoo Filter stores a small hash (fingerprint) of each item. Because the fingerprint can be read back out, you can check it and remove it — something fundamentally impossible in a Bloom Filter.

⊕

XOR Dual-Bucket Trick

Each item has two candidate buckets: b₁ = hash(item) % B and b₂ = b₁ ⊕ hash(fingerprint) % B. The XOR means: given only the fingerprint and one bucket index, you can always compute the other — enabling eviction without knowing the original item.

🥊

Cuckoo Eviction

If both candidate buckets for a new item are full, kick out a random occupant from one bucket and relocate it to its alternate bucket (computed via XOR). Repeat up to MAX_KICKS times. This is why it's called a "cuckoo" filter — like a cuckoo bird evicting eggs.

🗑️

Deletion Works!

To delete an item, compute its fingerprint and two candidate buckets, then scan both for the fingerprint and remove it. No rebuild, no false negatives for other items. The trade-off: if you insert an item twice, deleting once removes one copy — caller must be careful.

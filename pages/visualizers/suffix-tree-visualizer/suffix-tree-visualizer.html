🌲 The Trick That Makes This O(n)

1️⃣

Rule 1 — Implicit Leaf Extension

Every leaf edge uses a single shared "global end" reference instead of a fixed endpoint. When the string grows by one character, EVERY leaf extends automatically — zero extra work, this is what avoids re-touching every existing leaf.

2️⃣

Rule 2 — New Leaf or Split

If the current suffix doesn't continue along an existing edge, either a brand new leaf edge is created, or an existing edge is split and a new leaf branches off — the "interesting" case that actually creates tree structure.

3️⃣

Rule 3 — Show Stopper

If the next character already exists along the current edge, nothing is created — the algorithm just advances the active point and stops the current phase early. This early stop is the second half of what makes the whole algorithm linear.

🔗

Suffix Links

After creating an internal node via a split, a suffix link connects it to the NEXT internal node that will be created for the next suffix in the same phase — letting the algorithm jump there directly instead of walking from the root again.

📅 Weighted Interval Scheduling Explained

📋

Step 1: Sort by End Time

Sort all jobs in non-decreasing order of finish time. This ordering is what makes the DP recurrence work — every "earlier" job in our sorted list genuinely finishes earlier, so we can safely reference them as predecessors.

🔍

Step 2: Compute p(j) via Binary Search

For each job j, p(j) is the index of the latest job that finishes at or before job j's start time. Binary search on the sorted end-time array finds this in O(log n). If no job qualifies, p(j) = 0.

📊

Step 3: DP Recurrence

OPT(j) = max(OPT(j−1), profit(j) + OPT(p(j))). The first term skips job j (exclude). The second takes job j plus the best schedule using only jobs compatible with j (include). Base case: OPT(0) = 0.

↩️

Step 4: Traceback

Walk back from OPT(n). At each index j: if profit(j) + OPT(p(j)) > OPT(j−1), job j was included — add it and jump to p(j). Otherwise skip and go to j−1. Greedy-by-profit and greedy-by-end both fail on weighted inputs.

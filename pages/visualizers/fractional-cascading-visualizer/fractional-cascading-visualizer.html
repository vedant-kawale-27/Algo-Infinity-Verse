🌉 Why the Bridges Never Break

👻

Promoting Every Other Element

Each cascaded track is built from the list below it: take every other element and merge it into the track above, tagged as a "ghost." This roughly doubles each track's size but keeps total space linear across all K lists.

🔗

The Bridge Pointer

Every element in a cascaded track — original or ghost — stores a "bridge": the index of its own successor in the track below. This pointer is built once, in O(1) amortized per element, during construction.

🎯

Why One Binary Search Is Enough

Because every OTHER element was promoted, the position a bridge points to in the next track is never more than 1 slot away from the true answer — a tiny local scan, not a fresh binary search. This bounded drift is the entire trick.

🗺️

Where This Shows Up

Fractional cascading was invented for computational geometry problems — answering "which of these K line segments does a query ray cross first" across many slabs — and shows up in database range-query indexes and multi-layer map data structures.
